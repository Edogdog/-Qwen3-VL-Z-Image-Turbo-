# 【Intel AI PC创新应用征文】基于Qwen3-VL与Z-Image-Turbo的智能图像风格迁移应用

## 简介

本文介绍一款运行在 **Intel AI PC** 上的端侧图像风格迁移应用。核心创新在于：无需 GPU，仅利用 CPU 即可实现"上传任意风格参考图 → AI 理解风格语义 → 生成风格化图像"的完整流程。

**技术方案**：采用 **Qwen3-VL-4B** 作为视觉语言模型（VLM）提取风格特征，结合 **Z-Image-Turbo** 图像生成模型，通过 **OpenVINO** 推理引擎在本地完成全流程。解决了传统风格迁移依赖预设风格库、无法理解抽象美学概念的痛点，实现真正的零样本风格迁移。

**核心价值**：让普通用户在消费级硬件上，无需网络连接，即可获得专业级的 AI 风格化图像创作体验。

### 1.4 踩坑记录

#### 踩坑1: numpy 版本冲突
**问题**: requirements.txt 指定 `numpy<2.0`，但 Python 3.14 无预编译 wheel

**解决方案**: 手动安装 numpy==2.3.5 预编译版本

#### 踩坑2: GitHub 网络无法访问
**问题**: optimum-intel 和 diffusers 的 GitHub 源连接被重置

**解决方案**: 使用 PyPI 上的包代替

#### 踩坑3: Python 3.14 audioop 模块缺失
**问题**: Python 3.14 移除了内置 audioop，导致 pydub 导入失败

**解决方案**: 安装 audioop-lts 替代包

#### 踩坑4: Gradio 依赖版本冲突
**问题**: 部分依赖版本与 gradio 6.9.0 要求略有出入

**冲突详情**:

| 依赖项 | Gradio 要求 | 实际安装 | 是否影响 |
|--------|------------|---------|---------|
| aiofiles | >=22.0, <25.0 | 25.1.0 | 否 |
| gradio-client | ==2.3.0 | 2.4.0 | 否 |
| starlette | >=0.40.0, <1.0 | 1.0.0 | 否（已装fastapi） |

#### 踩坑5: optimum-intel 不支持 qwen3_vl 模型类型
**问题**: optimum-intel 1.27.0 的 MODEL_TYPE_TO_CLS_MAPPING 不包含 qwen3_vl

**错误信息**:
```
KeyError: 'qwen3_vl'
AttributeError: 'Qwen3VLVisionConfig' object has no attribute 'embed_dim'
```

**解决方案**:
1. 修改 config.json，添加 embed_dim
2. 使用 monkey-patch 映射到 qwen2_vl

```python
# 修改 config.json
import json
with open('Qwen3-VL-4B-Instruct-int4-ov/config.json') as f:
    config = json.load(f)
config['vision_config']['embed_dim'] = config['vision_config']['hidden_size']
with open('Qwen3-VL-4B-Instruct-int4-ov/config.json', 'w') as f:
    json.dump(config, f, indent=2)

# Monkey-patch
from optimum.intel.openvino.modeling_visual_language import MODEL_TYPE_TO_CLS_MAPPING, _OVQwen2VLForCausalLM
MODEL_TYPE_TO_CLS_MAPPING['qwen3_vl'] = _OVQwen2VLForCausalLM
```

#### 踩坑6: OVZImagePipeline 不可用
**问题**: optimum-intel 中没有 OVZImagePipeline 类

**解决方案**: 直接使用 OpenVINO Core 加载各组件

```python
import openvino as ov
core = ov.Core()
transformer = core.compile_model(core.read_model('transformer/openvino_model.xml'), 'CPU')
text_encoder = core.compile_model(core.read_model('text_encoder/openvino_model.xml'), 'CPU')
vae = core.compile_model(core.read_model('vae_decoder/openvino_model.xml'), 'CPU')
```

### 2.5 VAE 缩放因子问题修复

**问题现象**: 生成的图像呈现马赛克/像素化效果

**根本原因**: VAE 解码器需要使用配置文件中的缩放因子对 latent 进行预处理

**解决方案**: 在 VAE 解码前应用 `scaling_factor` 和 `shift_factor`

```python
# 加载 VAE 配置
vae_config = json.load(open(model_dir / 'vae_decoder' / 'config.json'))
scaling_factor = vae_config.get('scaling_factor', 0.3611)  # 0.3611
shift_factor = vae_config.get('shift_factor', 0.1159)       # 0.1159

# 解码前缩放
scaled_latents = (latents - shift_factor) / scaling_factor
```

## 三、应用场景说明

### 3.1 创新方向

本项目将 **视觉语言模型（VLM）** 与 **图像生成模型** 创新性地结合：

1. **核心创新点**
   - **语义级别的风格理解**：VLM 理解"什么是古典东方美学"等抽象概念
   - **零样本风格迁移**：无需预先定义风格类别，可处理任意风格参考图
   - **端侧部署**：基于 OpenVINO，在普通 Intel CPU 上高效运行

| 问题 | 解决方案 |
|------|----------|
| 专业设计技能门槛高 | 任何用户只需上传风格图即可获得风格化图像 |
